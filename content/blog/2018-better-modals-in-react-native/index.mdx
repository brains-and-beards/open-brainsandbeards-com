#### Customising pop-up animation

One note here, because I know it confuses some people. When we create an _interpolation_ we pass arrays for both input and output ranges. For a simple animation the input is usually `[0, 1]` and the output `[someValueX, someValueY]`. In that case it means that the value is **interpolated** between `someValueX` and `someValueY` **linearly**. So, in the middle of the animation (in case of a one-second-long animation, after half a second), the value will be exactly in the middle between them.

However, sometimes we donâ€™t want a linear change, but something more drastic. In that case we can pass more values to both `inputRange` and `outputRange` (we just need to make sure their lengths are the same). For example, if we want a one-second animation that changes opacity to quickly go from `1.0` to `0.5` in the first 100 ms and then slow down and linearly go to `0.1`, we can create the following interpolation:

```javascript {numberLines: true}

  inputRange: [0, 100, 1000],
  outputRange: [1.0, 0.5, 0.1],
});
```

In case of our animation, we have `inputRange: [thisSceneIndex â€” 1, thisSceneIndex, thisSceneIndex + 1]` and `thisSceneIndex` is the index of the screen thatâ€™s currently being pushed into view. So, the previous screen (which is being covered) will be `thisSceneIndex â€” 1`.

#### Code architecture

Okay, now we have a modal that looks and behaves really nice. Time to create an equally nice API for the developers to use.

Of course, you can choose whatever code architecture you want (itâ€™s JavaScript in the end), let me just share the API I like and find pleasant to work with:

```javascript {numberLines: true}
  readonly cancelLabel: string;
  readonly confirmAction: () => void;
  readonly confirmLabel: string;
  readonly text: string;
  readonly title: string;
}

  return {
    leftButtonHandler: cancelAction,
    leftButtonLabel: cancelLabel,
    rightButtonHandler: confirmAction,
    rightButtonLabel: confirmLabel,
    text,
    theme: 'warning',
    title,
  };
}
```
